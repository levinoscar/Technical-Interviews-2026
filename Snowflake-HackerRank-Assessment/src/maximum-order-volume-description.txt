Maximum Order Volume

A supermarket will receive calls with known start
times, durations, and order volumes. Only one
call can be handled at a time, and callers who
are not answered will not call back.

Determine which calls to answer to maximize
the total order volume.

Example
start = [10, 5, 15, 18, 30]
duration = [30, 12, 20, 35, 35]
volume = [50, 51, 20, 25, 10]

Call details:
 * Call 1: starts at t = 10, ends at t = 40, volume 
   = 50 
 * Call 2: starts at t = 5, ends at t = 17, volume 
   = 51
 * Call 3: starts at t = 15, ends at t = 35, volume 
   = 20
 * Call 4: starts at t = 18, ends at t = 53, 
   volume = 25
 * Call 5: starts at t = 30, ends at t = 65,
   volume = 10

THe first call completely overlaps calls 2 and 3,
and partially overlaps calls 4 and 5. By
answering calls 2 and 4, which do not overlap,
the maximum total order volume is 51 + 26 = 76.

Function Description
Complete the function phoneCalls in the editor
with the following parameter(s):
   int start[n]: the start times of each call
   int duration[n]: the durations of each call
   int volume[n]: the volumes of each other

Returns
  int: the maximum possible volume of colors
that can be received

Constraints
* 1 ≤ n ≤ 10^5
* 1 ≤ start[i] ≤ 10^9
* 1 ≤ duration[i] ≤ 10^9
* 1 ≤ volume[i] ≤ 10^3

## Input Format for Custom Testing

The first line contains an integer, n, the size of
the start array.
Each line i of the n subsequent lines (where 0 ≤ i
< n) contains an integer, start[i]
The next line contains repeats the integer, n, the
size of the duration array.
Each line i of the n subsequent lines (where 0 ≤ i
< n) contains an integer, duration[i]
The nest line repeats the integer, n, size of the
volume array.
Each line i of the subsequent lines (where 0 ≤ i
< n) contains an integer, volume[i]

## Explanation
On the following path as shown below: (1, 1) 
-> (2, 1) -> (3, 1) -> (3, 2) -> (3, 3) -> (3, 4) -> 
(4, 4), the strength of the path = 1, and the 
number of cells visited = 7.
Therefore, the answer = [1, 7].

blocked cells, the green stars represent the 
cells visited, and the blue arrows represent 
the path direction.

## Explanation
The calls happen in the intervals:
[1,3]
[2,4]
[4,5]
The first and third calls together make up the
order volume 4, and their intervals do not
intersect.
The first and second calls intersect, as do the
second and third calls.

Only one call from either of these pairs can be
serviced. The most efficient calls to answer are
the first and third, with a total volume of 4.
